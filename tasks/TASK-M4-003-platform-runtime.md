# TASK-M4-003：平台运行时、DPI、文件系统与故障恢复

**状态**：已完成（2026-07-16）；真实 Apple Silicon 系统集成回归归入 TASK-M4-005  
**依赖**：TASK-M4-001

## 目标

收敛 Windows/macOS 的路径、权限、高 DPI、休眠唤醒、网络切换、磁盘不足和原生错误提示。

## 范围与验收

- 平台差异只进入 `src/platform/` 或基础设施适配层。
- Retina/高 DPI 下画布坐标、选框和导出像素保持一致。
- 权限拒绝、离线、休眠恢复和磁盘不足不破坏已有图片或模型版本。
- 用户可恢复的错误提供明确操作，不泄露路径中的敏感信息。

## 预计修改文件

- `src/platform/`、`src/infrastructure/`、`src/ui/`、`tests/unit/`、`tests/integration/`

## 测试命令

```powershell
python -m pytest tests/unit tests/integration tests/ui -q
```

## 实际实现

- Qt 应用创建前固定高 DPI 小数缩放策略；画布继续使用逻辑坐标与原图像素坐标显式换算，Retina DPR=2 自动回归保持坐标往返一致。
- 新增应用状态和网络可达性监控。首次激活不重复刷新；从休眠/后台恢复或网络重新在线时合并事件，后台刷新图片限制、激活状态和模型。
- 图片限制、激活和模型刷新分别使用运行中门禁，连续恢复事件不会提交重复并发任务。
- 平台数据/缓存目录启动时执行短暂可写探测并立即删除；错误不显示用户目录。
- 图片导出按原始像素缓冲两倍且至少 16 MiB 预检空间；模型下载按剩余字节并保留 64 MiB，模型安装按完整模型大小并保留 64 MiB。
- 磁盘不足、只读目录、权限拒绝和离线存储错误映射为稳定中文提示，不拼接原生异常路径；临时写入或安装失败继续保留原图、既有导出文件和旧模型活动指针。
- 完整自动回归为 `281 passed`；Windows/macOS 真机休眠、Retina、网络切换和系统权限对话框仍在发布设备矩阵复核。
